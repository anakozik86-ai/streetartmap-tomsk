# Архитектура streetartmap

Документ фиксирует ключевые решения проекта. Изменения требуют отдельного PR
и обновления этого файла.

## Цели

- **Бесплатно** — без серверов, без платных сервисов в основном пути.
- **Масштабируемо** — модель данных одинаковая для любого города и не упирается в 50 точек.
- **Удобно для редакторов** — кастомная админка, визуальные редакторы точек и маршрутов.
- **Версионируемо** — все данные в git, history и audit trail бесплатно.

## Стек и почему

| Слой      | Решение                       | Почему                                                        |
| --------- | ----------------------------- | ------------------------------------------------------------- |
| Хостинг   | GitHub Pages                  | Бесплатно, без лимитов на трафик в разумных пределах          |
| Карта     | Leaflet + OSM                 | Open source, лёгкий, идеален для 50–500 точек                 |
| Тайлы     | CartoDB Voyager / Dark Matter | Не отвлекают от маркеров, есть тёмная и светлая версии        |
| Хранение  | JSON-файлы в репо             | Бесплатно, версионируется, мигрируется простым форком         |
| Фото      | Репо + jsDelivr CDN           | Бесплатный CDN поверх GitHub, без внешних аккаунтов           |
| Маршруты  | OSRM public (foot) + кэш      | Бесплатно, пешеходные, рантайм публички не зависит от сервиса |
| Auth      | GitHub Personal Access Token  | Без OAuth-прокси, аудит коммитов из коробки                   |
| Фреймворк | Preact + @preact/signals      | Маленький рантайм, React-совместимый API                      |
| Сборка    | Vite + TypeScript             | Стандарт индустрии, multi-entry, быстрый dev                  |
| Валидация | AJV в CI                      | Блокирует деплой при поломке данных                           |

## Модель данных

Шесть JSON-файлов в `data/`:

- `config.json` — глобальный конфиг города и сайта
- `categories.json` — 8 фиксированных категорий (мурал, граффити, ...)
- `collections.json` — фестивали и серии (объединены `type`)
- `authors.json` — авторы работ
- `points.json` — точки на карте
- `routes.json` — пеший маршруты-путеводители

Полные TS-типы в `src/shared/types/data.ts`.

### Ключевые принципы

- **Slug-идентификаторы** во всех сущностях; URL-friendly, читаемые в diff'ах.
- **Soft-delete** через `status: 'active' | 'archived'` для справочников;
  жёсткое удаление возможно только при отсутствии ссылок.
- **`status` и `state` ортогональны.** `status` — публикационный
  (`draft`/`published`/`archived`). `state` — физическое состояние работы
  (`intact`/`damaged`/`restored`/`painted_over`/`removed`/`unknown`).
- **Категория ≠ обводка.** Категория — иконка маркера. Цвет(а) обводки —
  принадлежность к коллекциям (фестиваль/серия). При нескольких коллекциях
  обводка — conic-gradient до 4 видимых секторов.
- **Утраченные работы** скрыты по умолчанию, переключатель «показать
  утраченные» делает их видимыми с пониженной непрозрачностью.
- **Авторы — справочник.** Свободная строка `author` исключена ради
  чистоты данных и страницы автора в будущем.

## Поток данных

```
┌─────────────────────────────────┐
│ Редактор в браузере             │
│ (admin SPA, login по PAT)       │
└────────────┬────────────────────┘
             │ GitHub Contents API
             │ (чтение и запись JSON, фото)
             ▼
┌─────────────────────────────────┐
│ Репозиторий GitHub              │
│ data/*.json + images/*          │
└────────────┬────────────────────┘
             │ push в main
             ▼
┌─────────────────────────────────┐
│ GitHub Actions: validate + build│
│ (block на ошибках валидации)    │
└────────────┬────────────────────┘
             │ deploy
             ▼
┌─────────────────────────────────┐
│ GitHub Pages — публичный сайт   │
│ читает /data/*.json через fetch │
└─────────────────────────────────┘
```

## Локализация

UI-строки вынесены в `locales/ru.json` с самого начала. Контент (названия,
описания) пока — простые строки. Миграция на `{ ru, en }` — одноразовый скрипт
с увеличением `schema_version`.

## Многогородье

Каждый город — отдельный репозиторий. Структура одинаковая, отличается только
содержимым `data/` и `data/config.json`. Инструкция: `docs/ADD-CITY.md`.
