# Инструкция для редакторов карты

> Этот документ — для людей, которые будут добавлять и редактировать точки,
> маршруты, авторов, фестивали и серии на карте streetartmap.
>
> _Полная инструкция появится после реализации админки. Пока — общие принципы._

## Что нужно для работы

1. Аккаунт на GitHub.
2. Доступ к репозиторию в роли Collaborator (попросите владельца репо).
3. Personal Access Token (PAT) с правами на этот репо.

## Как получить PAT

1. Зайдите на GitHub → Settings → Developer settings → Personal access tokens →
   **Fine-grained tokens** → **Generate new token**.
2. Repository access: **Only select repositories**, выберите репозиторий
   streetartmap.
3. Permissions → Repository permissions: **Contents: Read and write**.
4. Сгенерируйте токен и скопируйте его (он показывается один раз).
5. Откройте админку (`/admin/`) и вставьте токен в поле логина. Он сохранится
   в localStorage браузера, на сервере не хранится.

## Что вы будете делать

- **Добавлять и редактировать точки** — стрит-арт-объекты на карте.
- **Управлять справочниками** — категории (фиксированы), коллекции
  (фестивали и серии), авторы.
- **Собирать маршруты** из существующих точек.
- **Загружать фотографии** прямо из браузера. Они автоматически сжимаются
  и складываются в репозиторий.

## Принципы работы

- Все изменения **сразу попадают в репозиторий** под вашим GitHub-логином —
  это и есть аудит-лог.
- **Черновики** не показываются на публичной карте: ставите статус `draft`,
  спокойно работаете дальше, переключаете на `published`, когда готово.
- **Удалённое не пропадает.** Архив сохраняет историю; полное удаление
  возможно только если на запись нет ссылок.
- **Поломанный JSON не доедет до публики.** CI проверяет данные перед
  деплоем — если что-то не так, вам придёт уведомление в Pull Request.
