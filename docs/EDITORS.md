# Инструкция для редакторов карты

> Этот документ — для людей, которые добавляют и редактируют точки,
> маршруты, авторов, фестивали и серии на карте streetartmap.

## Что нужно для работы

1. Аккаунт на GitHub.
2. Доступ к репозиторию в роли Collaborator (попросите владельца репо).
3. Personal Access Token (PAT) с правами на этот репо.

## Как получить PAT

1. Зайдите на GitHub → Settings → Developer settings → Personal access tokens →
   **Fine-grained tokens** → **Generate new token**.
2. Repository access: **Only select repositories**, выберите репозиторий streetartmap.
3. Permissions → Repository permissions: **Contents: Read and write**.
4. Сгенерируйте токен и скопируйте его (он показывается один раз).
5. Откройте админку (`/admin/`) и вставьте токен в поле входа. Токен сохранится
   в localStorage браузера — на сервере не хранится.

## Принципы работы

- Все изменения **сразу попадают в репозиторий** под вашим GitHub-логином — это и есть аудит-лог.
- **Архивные записи** не показываются на публичной карте. Создавайте в архиве, доделывайте, переключайте на «Опубликовано» когда готово.
- **Удалённое не восстановить.** Перед удалением убедитесь, что на запись нет ссылок — иначе кнопка удаления заблокирована.
- **Поломанный JSON не доедет до публики.** CI проверяет данные перед деплоем.

## Точки и маршруты

### Создание

Нажмите **«+ Добавить»** в соответствующем разделе. Новая запись создаётся со статусом **«В архиве»** — она не видна на публичной карте. Заполните все поля и сохраните.

### Публикация

Каждая строка в таблице начинается с круглой кнопки-переключателя:

- **●** (заполненный круг) — запись опубликована, видна на карте.
- **○** (пустой круг) — запись в архиве, скрыта от посетителей.

Нажмите кнопку чтобы переключить статус. Изменение сразу сохраняется в репозиторий. Архивные строки в таблице отображаются полупрозрачно.

### Сортировка таблицы

Над таблицей — три режима сортировки:

- **Опубликованные вначале** — по умолчанию.
- **Архивные вначале** — удобно когда нужно разобрать накопившиеся черновики.
- **Свой порядок** — ручная расстановка кнопками ↑↓.

В режиме **«Свой порядок»** в каждой строке появляются кнопки ↑ и ↓. Перестановки не сохраняются автоматически — используйте кнопки над таблицей:

- **Сохранить порядок** — зафиксировать текущую расстановку (хранится в браузере).
- **Откатить** — вернуться к последнему сохранённому порядку.
- **Сбросить** — удалить сохранённый порядок, вернуться к «Опубликованные вначале».

Порядок сохраняется локально в браузере и не влияет на данные в репозитории.

### Редактирование

Нажмите **✎** в строке записи, чтобы открыть форму редактирования.

### Удаление

Нажмите **🗑** в строке записи.

- **Маршрут** удаляется после подтверждения.
- **Точку** нельзя удалить, пока она входит в какой-либо маршрут. Откроется список маршрутов с кликабельными ссылками — сначала уберите точку из маршрутов, затем удалите.

## Справочники: Категории, Коллекции, Авторы

Справочники не имеют статуса «опубликовано / архив» — все записи всегда активны.

### Создание и редактирование

Нажмите **«+ Добавить»** или **✎** в строке. Заполните форму и сохраните.

### Сортировка и свой порядок

Работает так же, как для точек и маршрутов: три режима над таблицей, кнопки ↑↓ в режиме «Свой порядок», Save / Revert / Reset.

### Удаление

Нажмите **🗑** в строке.

Перед удалением проверяется, есть ли точки, ссылающиеся на эту запись:

- Если есть — откроется список точек с кликабельными ссылками. Сначала снимите ссылки (измените категорию / уберите из коллекции / смените автора в точках), затем удалите запись.
- Если нет — откроется диалог подтверждения.

**Категории** нельзя создавать или переименовывать — список фиксированный (8 типов). Редактирование описания и иконки возможно.

## Фотографии

Загружаются прямо из браузера в форме точки. Автоматически сжимаются и сохраняются в репозитории. Указывайте подпись и авторство для каждой фотографии.

---

## Если что-то пошло не так

- **409 Conflict при сохранении** — кто-то другой сохранил этот файл одновременно. Обновите страницу и повторите.
- **Изменения не появляются на сайте** — деплой занимает 1–2 минуты после сохранения. Если прошло больше — проверьте вкладку Actions в репозитории на GitHub.
- **Токен перестал работать** — возможно, он истёк или был отозван. Сгенерируйте новый и войдите снова.
